# Проект: База данных "Кофейня"

## О проекте 
Данный проект представляет собой реляционную базу данных, разработанную для автоматизации работы кофейни. Система охватывает ключевые бизнес‑процессы: управление клиентами, обработку заказов, контроль товарных запасов, взаимодействие с поставщиками и организацию работы персонала. База данных реализована на PostgreSQL и обеспечивает целостность данных через систему внешних ключей и нормализованную структуру.

# Структура базы данных
## Сущности 
- *Clients* - содержит информацию о клиентах кофейни
- *Discount_card* - хранит данные о картах лояльности клиентов
- *Staff_members* - учёт персонала кофейни
- *Orders* - информация о закзах
- *Products* - каталог продукции кофейни
- *Products_in_order* - содержит состав заказов
- *Suppliers* - данные о поставщиках
- *Supply* - информация о поставках от поставщиков

## Связи 

- *Customers <> Discount_card* - один‑к‑одному.
Каждый клиент может иметь не более одной карты лояльности.
- *Customers <> Orders* - один‑ко‑многим.
Один клиент может оформлять множество заказов.
- *Staff_members <> Orders* - один‑ко‑многим.
Один сотрудник может обслуживать множество заказов.
- *Suppliers <> Supply* - один‑ко‑многим.
Один поставщик может осуществлять несколько поставок.
- *Products <> Supply* - один‑ко‑многим.
Один товар может поставляться от разных поставщиков.
- *Orders <> Products_in_order* - один‑ко‑многим.
Один заказ может включать несколько позиций товаров.
- *Products <> Products_in_order* - один‑ко‑многим.
Один товар может входить в состав разных заказов.

## Технологии 
- **PostgreSQL**
- **Ограничения целостности (Foreign key)**
- **ENUM-типы**
